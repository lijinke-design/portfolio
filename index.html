发现问题

内容创作的效率困境

小红书是中国最重要的消费决策平台之一，但专业的种草内容创作门槛高、耗时长，让大量中小卖家望而却步。

⏱️

时间成本高

一篇高质量种草笔记平均需要 2–4 小时：头脑风暴、撰写、修改、加标签，每个环节都耗神。

📉

转化率参差不齐

没有专业文案经验的卖家，笔记的点击率和加购率远低于平台均值，大量广告预算被浪费。

🔄

内容迭代慢

A/B 测试需要产出大量变体内容，人工撰写效率极低，错过内容红利窗口期。

设计过程

从需求到上线的
关键决策

1

用户痛点研究

调研小红书创作者的内容生产流程，发现"标题党焦虑"和"标签选择耗时"是最高频的痛点。将 AI 功能聚焦在这两个节点，而非追求全功能。

2

最小可行 AI 交互设计

拒绝了"对话式 AI"的方案——用户在创作态不想思考下一步问什么。转而设计"划词 + 意图猜测"模式：选中文字即触发工具栏，推荐动作自动高亮，减少用户决策次数。

3

采用 Liquid Glass 设计语言

基于 WWDC25 Apple 新设计规范，将导航层变为浮动透明玻璃，让内容成为视觉焦点。同时制定渐进适配方案，确保 iOS 17 用户也有接近原生的体验，而不是降级到普通 Material。

4

多 Agent 并行开发

项目采用"项目宪法"约定分工边界，四个开发 Agent 并行完成 UI 层、数据层、网络层和各功能模块，通过共享 API 契约（Repository、AuthStore、GeneratorProtocol）保证接口一致。

5

向前兼容架构

AI 调用层通过 GeneratorProtocol 抽象，可无缝切换 Mock / 真实 LLM / 不同模型供应商；Liquid Glass 通过编译期宏进行版本门控，未来 iOS 26 普及后可一行代码切换原生实现。
